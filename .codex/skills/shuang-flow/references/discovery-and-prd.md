# 想法澄清与 PRD

当用户还在“想法/方向/需求收敛”阶段时读取本文件。

## Vibe Coding 全链路

1. 想法
2. 调研
3. PRD
4. 技术架构设计
5. UI 原型设计
6. 前端代码
7. 后端代码
8. 系统链条
9. 部署上线

## 想法阶段要澄清什么

- 满足哪个用户的什么需求。
- 用户痛点来自访谈、调研、竞品分析，还是用户直接提出。
- 从调研文件收敛到 PRD 前，哪些判断还缺证据。

## 苏格拉底式 11 问

价值定义：

1. 目标与非目标。
2. 用户画像与反画像。
3. 核心用户故事。

功能边界：

4. MVP 与范围外。
5. 边界条件。

验收标准：

6. 可数字化的 NFR。
7. AC 验收用例。
8. MoSCoW 优先级。
9. 北极星指标与关停线。

风险识别：

10. 开放问题。
11. 依赖与约束。

## PRD Writer 模式

如果项目级 `prd-writer` skill 已安装，可使用：

```text
基于 brainstorming 的收敛结果和 specs/research/ 下的调研文件，
用 prd-writer Skill 帮我写 PRD（产出到 specs/prd.md）。
```

如果没有该 skill，就直接从 evidence 写 `specs/prd.md`，并列出缺失信息。
