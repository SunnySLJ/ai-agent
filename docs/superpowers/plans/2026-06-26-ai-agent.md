> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a complete work-folder career project for a Java engineer transitioning into AI Agent/RAG engineering.

**Architecture:** The project is documentation-first. It references the parent `../../agent/` course corpus without mutating it, and creates a self-contained learning, portfolio, interview, and career-growth system under `work/ai-agent/`.

**Tech Stack:** Markdown, GitHub, local course corpus, BOSS job-market review, Python Agent/RAG, FastAPI, LangGraph/LangChain, LlamaIndex, Java/Spring Boot business tools, MCP, OpenAI Agents SDK concepts, Docker/K8s.

## Global Constraints

- Do not modify `../../agent/`.
- Keep all new files under `work/ai-agent/`.
- Position the user as Java backend + AI Agent/RAG application engineer.
- Design for Hangzhou 20K target jobs first, with growth toward 20-40K.
- Every document must support learning, portfolio output, interview readiness, or long-term growth.
