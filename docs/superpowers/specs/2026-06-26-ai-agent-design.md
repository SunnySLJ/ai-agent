# AI Agent Engineer Career Design

## 背景

用户是 30 岁、5 年 Java 工程师，当前诉求是重新找工作，目标方向是 AI Agent 工程师，期望先用 1 个月入门并形成求职竞争力，地点以杭州为主，薪资目标先看 20K 左右，并希望 35 岁前形成持续成长路径。

本地已有大量课程资料，集中在父级 `../../agent/`，覆盖 LLM API、本地部署、Agent、LangChain、RAG、LlamaIndex、Agent Skills、Memory、Context、评估、Docker/K8s、OpenClaw、Harness、Claude Code 和综合案例。

## 核心定位

项目定位不是“泛 AI 学习仓库”，而是“岗位倒推的 AI Agent 工程师求职作战仓库”。

优先方向：

- Java 后端 + AI Agent/RAG 应用工程师。
- 企业知识库、智能客服、文档审核、数据分析、内部工具 Agent。
- Agent 编排、工具调用、RAG 检索、评估观测、服务化部署。

不优先方向：

- 纯算法研究。
- 大模型预训练。
- 大规模 GPU 训练平台。
- 只会调 Prompt 的低工程含量岗位。

## 项目结构

所有新增内容放在 `work/ai-agent/`：

- `README.md`：项目入口。
- `AGENTS.md`：Codex/Agent 执行规则。
- `CLAUDE.md`：Claude 使用规则。
- `docs/`：岗位、技术栈、计划、资料映射、面试和成长路线。
- `portfolio/`：作品集项目说明。
- `logs/daily/`：每日学习和求职日志。

父级 `../../agent/` 只作为资料来源，不改动。

## 成功标准

30 天后至少具备：

- 能解释 AI Agent、RAG、工具调用、Memory、Context、评估和部署的核心概念。
- 能拿出一个 Java 主导的 AI Agent/RAG 项目设计与实现记录。
- 能讲清楚一个 Agent 系统从用户问题到检索、工具调用、生成、评估、日志追踪的完整链路。
- 能用 BOSS 直聘岗位关键词反向调整简历和学习优先级。
- 能每天持续记录学习、作品和面试表达。

## 风险控制

- 课程资料很多，必须按岗位倒推筛选，避免平刷。
- 一个月不能追求“全栈 AI 专家”，必须先形成可面试的工程闭环。
- 简历不能夸大算法能力，必须强调 Java 工程化、RAG、Agent 落地和系统可靠性。
- BOSS 直聘岗位变化快，岗位分析文档必须标注为快照，并要求定期刷新。
